restrict_energy_range(energy_range=None, method=u'binned')[source]¶

Restrict to a given energy range

If no energy range is given, it will be extracted from the PHA header. Tow methods are available . Unbinned method: The new counts vectors are created from the list of on and off events. Therefore this list must be saved in the meta info. Binned method: The counts vectors are taken as a basis for the energy range restriction. Only bins that are entirely contained in the desired energy range are copied.

Parameters:

energy_range : EnergyBounds, optional

Desired energy range

method : str {‘unbinned’, ‘binned’}

Use unbinned on list / binned on vector
Returns:

obs : SpectrumObservation

Spectrum observation in desired energy range
classmethod stack_observation_list(obs_list, group_id=None)[source]¶

Create SpectrumObservations from list

Observation stacking is implemented as follows Averaged exposure ratio between ON and OFF regions, arf and rmf \(\alpha_{\mathrm{tot}}\) for all observations is calculated as .. math:: alpha_{mathrm{tot}} = frac{sum_{i}alpha_i cdot N_i}{sum_{i} N_i} .. math:: arf_{mathrm{tot}} = frac{sum_{i}arf_i cdot livetime_i}{sum_{i} livetime_i} .. math:: rmf_{mathrm{tot}} = frac{sum_{i}rmf_i cdot arf_i cdot livetime_i}{sum_{i} arf_i cdot livetime_i}

Parameters:

obs_list : list of SpectrumObservations

Observations to stack

group_id : int, optional

ID for stacked observations
